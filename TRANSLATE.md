# TRANSLATE.md — Localizing Match Three to Chinese

This file documents every piece of visible text in the game and exactly how to change each one to Chinese (Simplified). All changes are made inside the single `match3.html` file.

---

There are 10 distinct pieces of user-facing text, spread across the HTML markup and JavaScript. They are listed below with their location, current English value, and the Chinese replacement.

## Required CSS change — add Chinese font support

DM Mono and DM Sans do not contain Chinese glyphs. You must add a Chinese fallback font so that Chinese characters render correctly instead of falling back to the browser default.

In the `<link>` tag section of `<head>`, add a second Google Fonts request for Noto Sans SC (Simplified Chinese):

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@400;500&display=swap" rel="stylesheet">
```

Then update the two font-family declarations in the CSS that affect visible text:

**Body (DM Sans is used for general text)** — find:
```css
font-family: 'DM Sans', sans-serif;
```
Replace with:
```css
font-family: 'DM Sans', 'Noto Sans SC', sans-serif;
```

**DM Mono is used for labels, stat values, buttons, and the message area.** Chinese characters do not exist in monospace form in this font stack, so for any element using `font-family: 'DM Mono', monospace` that now displays Chinese text, add the fallback:
```css
font-family: 'DM Mono', 'Noto Sans SC', monospace;
```

The specific CSS rules that need this fallback added are the ones applied to: `h1`, `.stat-label`, `.stat-value`, `#msg`, `#new-btn`, `#overlay-title`, `#overlay-sub`, and `.overlay-btn`.

## What does NOT need to change

- The numbers displayed in `#score-val` and `#moves-val` are generated by JavaScript and are already numerals — no change needed.
- The gem emoji (🔴🟠🟡🟢🔵🟣) are Unicode and render identically in all locales — no change needed.
- All CSS class names, IDs, JavaScript function names, and variable names are code — do not translate them.
